# Telegram-бот для постановки задач сотрудникам с email-уведомлениями

## Описание проекта

Данный проект представляет собой Telegram-бота для постановки и распределения задач сотрудникам с автоматической отправкой уведомлений по электронной почте и системой напоминаний.

Бот использует обработку естественного языка через LLM (OpenAI API или локальный парсер) для извлечения структуры задачи из текстового сообщения пользователя.

---

## Основной функционал

- Авторизация пользователя по паролю
- Ввод задачи в свободной текстовой форме
- Обработка текста задачи с помощью LLM:
  - выделение адресатов
  - извлечение текста задачи
  - определение дедлайна
- Подтверждение задачи перед отправкой
- Отправка задач на email через SMTP
- Система напоминаний:
  - за 1 день до дедлайна
  - в день дедлайна
  - после просрочки (однократно)
- Отмена задачи с уведомлением адресатов
- Хранение задач в базе данных (SQLite или PostgreSQL через SQLAlchemy)

---

## Архитектура проекта

Проект состоит из следующих компонентов:

- Telegram Bot (aiogram 3) — обработка сообщений и взаимодействие с пользователем
- LLM Adapter — извлечение структуры задачи из текста (OpenAI API или stub-реализация)
- Database (SQLAlchemy) — хранение пользователей, задач и статусов
- SMTP Service — отправка email-уведомлений
- Scheduler (APScheduler) — планирование напоминаний
- Configuration (.env) — настройки окружения и секреты

---

## Логика работы системы

1. Пользователь запускает бота командой `/start`
2. Вводит пароль для авторизации
3. Отправляет текст задачи
4. LLM извлекает:
   - список получателей
   - описание задачи
   - дедлайн
5. Бот запрашивает подтверждение:
   - текст задачи
   - дедлайн
   - список адресатов
6. Пользователь выбирает:
   - ОТПРАВИТЬ — задача сохраняется и отправляется на email
   - ОТМЕНА — задача удаляется
7. После отправки:
   - выполняется отправка email каждому получателю
   - запускается система напоминаний

---

## Система напоминаний

Используется APScheduler с интервалом проверки (примерно каждые 15 минут):

- За 1 день до дедлайна
- В день дедлайна
- После просрочки (один раз, на следующий день)

Все уведомления отправляются по email.

---

## Структура базы данных

Основные таблицы:

- `users` — пользователи бота
- `recipients` — справочник сотрудников (ФИО и email)
- `tasks` — задачи:
  - текст задачи
  - дедлайн (UTC)
  - статус
  - список получателей
  - флаги отправленных напоминаний

## Конфигурация (.env)

Создайте файл `.env` на основе `.env.example`:

```env
BOT_TOKEN=your_telegram_bot_token

DB_URL=postgresql+asyncpg://user:password@host:5432/dbname

ADMIN_PASSWORD=your_password

SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=email@example.com
SMTP_PASSWORD=password
SMTP_FROM=email@example.com

SMTP_USE_TLS=true
SMTP_USE_SSL=false

LLM_PROVIDER=stub

OPENAI_API_KEY=your_key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
