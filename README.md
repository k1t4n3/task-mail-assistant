# Помощник постановки задач по почте (Telegram + Python)

Канал взаимодействия: **Telegram-бот** (aiogram 3).  
Голосовой ввод **не используется** (по выбору проекта — только текст; ASR не подключается).

## Пользовательский путь (по ТЗ)

1. **Авторизация** — после `/start` вводится пароль из `.env` (`ADMIN_PASSWORD`). При ошибке — повторный ввод.
2. **Описание задачи** — текстом. Можно прислать уточнение **ещё одним текстовым сообщением** до команды отправки.
3. **Подтверждение** — бот показывает: формулировку, срок, список адресатов (ФИО + email из справочника `recipients`). Команды: **`ОТПРАВИТЬ`** / **`ОТМЕНА`** (отмена без писем адресатам).
4. **Отправка** — на **каждый** email адресата уходит письмо «постановка задачи» (SMTP).
5. **Напоминания по почте** (включено по умолчанию) — правило «после просрочки»: **одно письмо в первый календарный день после дедлайна** (см. ниже). Также: **за день до** дедлайна и **в день дедлайна**. Письмо о **создании задачи** отправляется сразу при `ОТПРАВИТЬ`.
6. **Отмена задачи** — в состоянии ввода задачи напишите **`ОТМЕНИТЬ ЗАДАЧУ`**, выберите номер из списка (до **25** активных). Задача снимается, **всем адресатам** уходит письмо «задача отменена».

### Несколько адресатов в одном сообщении

Система стремится извлечь **несколько фамилий** (например: «Сидорову и Петровой — …»). Это **одна задача** на **нескольких** адресатов (одна формулировка, один срок).

### Часовой пояс и даты

Сроки для напоминаний и отображения переводятся в зону из `APP_TIMEZONE` (по умолчанию `Europe/Moscow`). Дедлайн в БД хранится с учётом временной зоны (UTC).

## LLM

- `LLM_PROVIDER=stub` — локальный эвристический парсер (`llm_adapter.py`).
- `LLM_PROVIDER=openai` — запрос к **OpenAI-совместимому** Chat Completions API:
  - `OPENAI_API_KEY`
  - `OPENAI_BASE_URL` (по умолчанию `https://api.openai.com/v1`)
  - `OPENAI_MODEL` (например `gpt-4o-mini`)

При ошибке сети/ответа выполняется **откат на stub**.

## Справочник сотрудников (`recipients`)

Таблица `recipients`: поле `surname` — **нормализованная фамилия** (нижний регистр, без лишних символов), `full_name`, `email`.  
Пример первичного наполнения — `storage_init.sql` (выполнить в pgAdmin **после первого запуска бота**, когда таблицы уже созданы).

## Установка и запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

На Windows для зон вроде `Europe/Moscow` в `zoneinfo` используется пакет **`tzdata`** (он уже указан в `requirements.txt`).

Создайте `.env` по образцу `.env.example`, затем:

```bash
python main.py
```

## Переменные окружения (`.env`)

| Переменная | Назначение |
|------------|------------|
| `BOT_TOKEN` | токен Telegram-бота |
| `DB_URL` | `postgresql+asyncpg://USER:PASS@host:5432/dbname` |
| `ADMIN_PASSWORD` | пароль входа в бота |
| `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM` | отправка почты |
| `SMTP_USE_TLS` | STARTTLS для порта 587 (`true`/`false`) |
| `SMTP_USE_SSL` | implicit SSL для порта 465 (`true`/`false`) |
| `LLM_PROVIDER` | `stub` или `openai` |
| `OPENAI_*` | при `openai` |
| `APP_TIMEZONE` | IANA-имя зоны, напр. `Europe/Moscow` |
| `REMINDERS_ENABLED` | `true`/`false` — включить фоновые напоминания |

Секреты **не коммитить**; в репозитории только `.env.example`.

## Миграция БД (если таблица `tasks` уже была без колонки)

Если проект запускался раньше, добавьте колонку вручную:

```sql
ALTER TABLE tasks ADD COLUMN IF NOT EXISTS reminders_sent JSONB NOT NULL DEFAULT '{"assignment": false, "day_before": false, "day_of": false, "overdue": false}'::jsonb;
```

(Если у вас тип `JSON` вместо `JSONB` — замените тип на совместимый с вашей схемой.)

## Напоминания (детализация)

Планировщик: **APScheduler**, интервал **15 минут** (`scheduler.py`).

- **Создание задачи**: письмо сразу при `ОТПРАВИТЬ` (флаг `assignment`).
- **За день до**: в календарный день перед датой дедлайна (в `APP_TIMEZONE`).
- **В день дедлайна**: в календарный день дедлайна.
- **После просрочки**: **ровно один раз** — в **следующий** календарный день после даты дедлайна (если задача всё ещё со статусом `sent`).
